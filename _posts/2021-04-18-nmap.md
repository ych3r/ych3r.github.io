> 最近用 nmap 扫描导致自己 ip 被目标封了，还好没被溯源。所以梳理一下 nmap 使用方法~

[nmap 官网](https://nmap.org/)

一次成功的网络连接与两个端口有关：一个是服务器开放的监听端口，另一个是本机随机选择的端口（通常数值很大）。

一些固定的端口：
* HTTP Web服务 - 80
* HTTPS Web服务 - 443
* Windows NETBIOS - 139
* SMB - 445

## 常用参数

* -sT: TCP 连接扫描
* -sS: SYN 半开放扫描
* -sU: UDP 扫描
* -sN: TCP Null
* -sF: TCP FIN
* -sX: TCP Xmas

### -sT: TCP 扫描

在此之前需要了解 TCP 三次握手。

#### 如何判断端口开放的呢？

nmap 发送 SYN 请求后：
* 如果目标端口关闭，它将返回 **RST**，由此 nmap 可以判断此端口关闭。
* 如果目标端口开放，它将返回 **SYN/ACK**，nmap 判断此端口开放，并发送 ACK 完成三次握手。
* 如果目标端口开放，但中间有防火墙（很多防火墙会 drop 请求），导致没有回应，nmap 由此判断此端口 filtered。

### -sS: SYN 扫描

和 TCP 扫描不同，SYN 扫描更像是 **half-open** 的扫描。它在先发送 SYN，收到 SYN/ACK 后并不发送 ACK 完成三次握手，而是发送 RST。

这样做的好处在于：
* 绕过一些古老的入侵检测系统，它们会检测三次握手。
* 通常不会被在监听的应用记录，因为没有建立完整连接。
* 比 TCP 扫描快。

坏处在于：
* 需要 sudo 权限，因为需要构建 raw 数据包。
* 有可能会把不稳定的服务搞坏。

如果在使用 nmap 时加上 sudo，将默认 SYN 扫描；反之 TCP 扫描。

#### 如何判断端口开放的呢？

除了处理开放端口时不一样，其它和 TCP 扫描一样。

### -sU: UDP 扫描

众所周知，UDP 是 stateless（无状态的），所以没有三次握手以确保消息传到，它只关心速度（并不是说扫描会快）。

#### 如何判断端口开放的呢？

UDP 不会收到回复，所以很难判断端口是否真的开放，扫描时间会很长。如果关闭了，目标将返回一个 ICMP 数据包。

### -sN -sF -sX: NULL FIN Xmas

并不常用，但适合用来伪装。因为有些 IDS 检测 SYN，所以用这几种方法来绕过防火墙。然而并不代表好用。

### -sn: ICMP 扫描

用来判断 ip 段内目标是否存在。

## NSE 脚本

* safe: 不影响目标
* intrusive: 可能会影响目标
* vuln: 扫描 vulnerabilities
* exploit: 尝试破解
* auth: 尝试绕过认证
* brute: 尝试暴力破解
* discovery: 尝试查询更多信息

```
e.g.

--script vuln
--script=http-fileupload-exploiter
--script=smb-enum-users,smb-enum-shares
```

#### 如何找到想使用的脚本？

去 **/usr/share/nmap/scripts/** 路径下找。

## 绕过防火墙

[官方文档 - 绕过检测](https://nmap.org/book/man-bypass-firewalls-ids.html)

nmap 在扫描前会先 ping 一下判断目标是否存活，然而 Windows 默认的防火墙会阻止所有 ICMP 数据包。可以使用 **-Pn** 参数省略 ping。

使用 **-f** fragment 将数据包切分，绕过检测。还可以加上 **--mtu <number>** 控制切分后的数据包大小（8的倍数）。

使用 **--scan-delay \<time>ms** 增加数据发送间的延迟，类似“心跳”，可用来绕过基于时间的防火墙或 IDS。

使用 **--badsum** 来生成无效数据包，虽然 TCP/IP 协议栈不接受它，但防火墙会自动回应，由此判断是否存在防火墙或 IDS。
